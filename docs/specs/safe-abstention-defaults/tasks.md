## Scoping decisions (locked 2026-07-17)

The open questions
([`requirements.md` "Open questions"](requirements.md#open-questions-for-scoping)
/ [`design.md` §6](design.md#6-open-questions-for-scoping)), answered
from the M1/M2 measurements + repo precedents:

1. **Bundled-default API surface — C1 via the bundled-corpus wiring**
   (design §3 C1, option 1): the calibrated threshold rides with the
   bundled attune-help corpus / pipeline factory, so `RagPipeline()`
   gets it while raw `KeywordRetriever()` keeps the class default 2.0
   (escape hatch preserved). Exact mechanism finalized in M3.
   **Data wrinkle (M1):** calibrated T=5 drops exactly one gate query
   (`SAST scan my repository`, top-1 = 3.75), violating R1. Path:
   alias remediation first (`aliases_override.json` fast path or
   upstream attune-help frontmatter — run the full baseline
   diagnostic before committing aliases), then T=5. If the alias fix
   is declined, fall back to the R1-clean **T=3** (41.7% negatives
   abstained — ~5× the current default's 8.3%, still short of the 92%
   target).
2. **`### Changed`, no override — Q2 is moot:** the Phase 4 freeze
   mechanism was retired 2026-06-10 (#181). The behavior change ships
   as `### Changed` in a minor release, per the #120 precedent.
3. **C3 verdict: FAILS — does not ship** (M2, measurements.md):
   no variant (per-token, gap, combination) achieves useful abstention
   inside the ≥95%-legit-on-both-corpora bar. Root cause: corpus_b's
   weakest legit query (per-token ≈0.21) scores *below* attune-help's
   negative range (≥0.25) — the cross-corpus overlap design §2
   predicted. BYO-without-data keeps the conservative default + a loud
   doc pointer to `--calibrate-abstention`.
4. **Auto-calibration entry point — first-class library API**
   (`RagPipeline.calibrated(corpus, queries, negatives)` classmethod,
   reusing `_calibrate_abstention`), consumed by the
   `user-corpus-onboarding` harness (R4 single source of truth).
   Since C3 failed, calibration is the *only* honest BYO path — it
   belongs in the library, not just the harness. Signature finalized
   at M4.
5. **POLICY §7 stays factual/advisory:** states that the bundled
   corpus ships a calibrated abstention default and names the
   calibration script; **no numeric floor commitment** (keyword-score
   metrics are composition-dependent — corpus changes move them; per
   the POLICY-commitment feedback rule and risk §3).
6. **Cross-tier abstention: keyword-tier-only at v1.0.0.** The CLI's
   explicit rejection of `--min-score` with non-keyword retrievers
   (2026-06-10 audit remediation) remains the v1.0.0 contract; docs
   state the protection loss on tier upgrade. Extending abstention to
   hybrid/transformer tiers needs a confidence signal of its own →
   deferred to `confidence-gated-retrieval` / v1.1.0+.

- The bundled exemplar is safe-by-default (false-answer rate at the
  measured target, zero legit-recall regression on the gate set).
- BYO corpora have an honest path (auto-calibrate or conservative
  floor) — no unsafe universal constant shipped.
- The default posture has a single source of truth shared with
  `user-corpus-onboarding`.
- All decisions and the calibration are reproducible from committed
  scripts + inputs.
